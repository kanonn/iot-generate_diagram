这个情报**非常重要**！既然你们在申请资源时就已经提交了这 3 个邮箱，那我们的沟通策略就要从“请帮我建”变成**“请帮我查一下状态，并只给发送权限”**。

### 💡 为什么列表现在看起来是“空的”？

在 AWS SNS 中，当你添加了一个邮箱订阅后，它的状态一开始会是 **`PendingConfirmation`（等待确认）**。
如果你们这 3 个邮箱**一直没有去点击 AWS 发来的那封确认邮件**，或者印度团队建的时候漏掉了，亦或者你当前的账号没有 `List` 的权限，它看起来就会是空的。

但无论列表是不是空的，**你的 Java 代码都不应该去尝试 `Subscribe` 或 `List`，它只需要无脑 `Publish` 就可以了。**

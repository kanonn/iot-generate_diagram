周一效率挺高啊！既然后端已经搞定了，前端 React 的 Docker 镜像创建其实非常标准化。

虽然 React 只是前端，但把它塞进 Docker 还是得讲究一下“身材”（镜像大小）和“性能”。最专业的做法是**多阶段构建 (Multi-stage Build)**：先用 Node 镜像把代码编译出来，再把编译后的静态文件丢进轻量级的 Nginx 镜像里。

这里有一个你可以直接拿去用的 `Dockerfile`：

### 📄 1. 编写 Dockerfile

在你的 React 项目根目录下创建一个名为 `Dockerfile` 的文件：

```dockerfile
# --- 第一阶段：构建 (Build) ---
FROM node:18-alpine AS build-stage

WORKDIR /app

# 先复制 package.json 提高缓存效率
COPY package*.json ./
RUN npm install

# 复制所有代码并打包
COPY . .
RUN npm run build

# --- 第二阶段：生产环境 (Production) ---
FROM nginx:stable-alpine

# 将第一阶段生成的 build 文件夹内容复制到 Nginx 的默认静态资源目录
COPY --from=build-stage /app/build /usr/share/nginx/html

# ⚠️ 这一步很重要：把自定义的 Nginx 配置考进去，解决 React 路由 404 问题
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]

```

---

### 📄 2. 编写 nginx.conf (关键点)

由于 React 是单页应用（SPA），如果用户刷新页面或者直接输入子路径，Nginx 会报 404。你需要一个简单的配置文件来处理路由跳转。

在根目录下创建 `nginx.conf`：

```nginx
server {
    listen 80;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
        # 核心：找不到文件时，统一转发到 index.html
        try_files $uri $uri/ /index.html;
    }

    # 如果有必要，你可以在这里加一些缓存控制
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }
}

```

---

### 🚀 3. 打包并推送到 ECR

一切准备就绪后，执行以下命令：

1. **打包镜像**：
```bash
docker build -t dproto-qa-frontend .

```


2. **标记镜像（Tag）**：
```bash
docker tag dproto-qa-frontend:latest 733155003398.dkr.ecr.ap-northeast-1.amazonaws.com/dproto/qa/frontend:latest

```


3. **推送镜像**：
```bash

```
